# 《AI替代危机》详细设计

> 基于用户的深度问题：玩家怎么提升？对手怎么变换？危机是什么？玩家玩的是什么？

---

## 一、核心问题解答

### 玩家玩的是什么？

```
你是一个普通打工人，AI正在快速进化。
每过一段时间，AI就能做更多事情。
你要在AI完全替代你之前，找到自己的出路。

核心体验：
- 焦虑感（AI越来越强）
- 策略选择（学什么、做什么）
- 生存压力（不能被替代）
- 成就感（找到出路）
```

### 危机是什么？

```
危机 = AI的能力不断提升 × 你的价值不断降低

具体表现：
1. 工作内容被AI替代
   - 以前你写的代码 → AI能写了
   - 以前你做的图 → AI能做了
   - 以前你写的文案 → AI能写了

2. 收入下降
   - 同样的工作，AI更便宜
   - 老板开始裁员
   - 只能接更便宜的单

3. 时间压力
   - AI进化速度很快
   - 你没有无限时间学习
   - 必须做出选择

4. 心理压力
   - 焦虑（会不会被替代）
   - 迷茫（该学什么）
   - 自我怀疑（还能做什么）
```

## 二、玩家怎么提升？

### 局内成长（每局内）

```
1. 技能学习
   - 选择学习某个技能
   - 技能有熟练度
   - 熟练度影响收入

2. 工具掌握
   - 学会使用AI工具
   - 用AI辅助自己工作
   - 提高效率

3. 项目经验
   - 完成项目获得经验
   - 经验影响口碑
   - 口碑影响接单

4. 人脉积累
   - 认识客户
   - 认识同行
   - 获得推荐
```

### 局外成长（跨局继承）

```
1. 技能点
   - 每局获得技能点
   - 技能点可以永久保留
   - 下一局开局更强

2. 解锁技能
   - 完成特定成就解锁新技能
   - 例如：第一次用AI赚钱 → 解锁"AI协作"
   - 新技能在下一局可用

3. 解锁职业
   - 完成特定条件解锁新职业
   - 例如：程序员 → AI训练师
   - 不同职业有不同玩法

4. 解锁结局
   - 达成不同结局
   - 解锁新的开局选项
```

## 三、对手（AI）怎么变换？

### AI的进化阶段

```
阶段1：AI入门期（第1-3回合）
- AI能做简单重复工作
- 例如：简单翻译、基础文案
- 影响：低端工作机会减少

阶段2：AI成长期（第4-6回合）
- AI能做中等复杂工作
- 例如：代码生成、图片生成
- 影响：更多工作被替代

阶段3：AI成熟期（第7-9回合）
- AI能做复杂工作
- 例如：项目规划、数据分析
- 影响：大部分工作被替代

阶段4：AI超越期（第10+回合）
- AI能做人做不到的事
- 例如：24小时不休息、同时处理100个项目
- 影响：人类很难竞争
```

### AI的随机变化（肉鸽要素）

```
每局AI的进化路径不同：

路径A：效率优先
- AI先提升速度
- 能更快完成工作
- 对手特点：快

路径B：质量优先
- AI先提升质量
- 能做更好的工作
- 对手特点：好

路径C：成本优先
- AI先降低成本
- 价格更便宜
- 对手特点：便宜

路径D：全能发展
- AI全面发展
- 没有明显弱点
- 对手特点：全面

随机事件：
- 某公司大量采用AI → 某行业突然被冲击
- 新AI工具发布 → 某技能突然贬值
- AI出现Bug → 暂时性机会
- AI监管政策 → 某些领域暂时安全
```

## 四、核心循环

### 每回合流程

```
1. AI进化
   - AI获得新能力
   - 某些工作被替代
   - 收入可能下降

2. 市场变化
   - 某些技能需求上升/下降
   - 某些工作机会出现/消失
   - 价格波动

3. 玩家行动（选择一个）
   - 学习新技能
   - 接项目赚钱
   - 用AI辅助工作
   - 转型探索
   - 休息恢复

4. 结算
   - 收入变化
   - 技能提升
   - 压力变化

5. 判定
   - 是否被替代
   - 是否找到出路
   - 是否崩溃
```

## 五、具体设计

### 技能系统

```
技能树：

基础技能（开局就有）
├── 文案写作
├── 基础编程
├── 图片处理
└── 数据整理

进阶技能（需要学习）
├── AI协作 ← 学会用AI辅助自己
├── 创意策划 ← AI不太擅长的
├── 人际沟通 ← AI做不到的
├── 项目管理 ← 综合能力
└── AI训练 ← 帮AI变得更好

特殊技能（需要解锁）
├── AI工具开发 ← 做AI的人
├── 跨界整合 ← AI做不到的
└── 体验设计 ← 需要人类感知
```

### 职业路径

```
初始职业：
- 程序员 → 最容易被AI替代
- 设计师 → 部分被AI替代
- 文案 → 部分被AI替代
- 数据分析 → 大部分被AI替代

转型职业（需要条件）：
- AI训练师 → 帮AI进化
- AI工具使用者 → 用AI赚钱
- 创意策划 → AI不擅长
- 用户体验设计 → 需要人类感知
- 自媒体 → 个人IP

终极职业（需要达成条件）：
- AI公司老板 → 雇佣AI
- 数字游民 → 自由生活
- 传统手艺人 → AI无法替代
```

### 结局系统

```
失败结局：
- 被替代 → 失业，AI完全取代你的工作
- 崩溃 → 心理压力太大，放弃
- 穷困 → 收入太低，无法生存

普通结局：
- 勉强生存 → 找到AI不擅长的小众领域
- 打工人 → 成为会用AI的普通员工
- 转型成功 → 换到相对安全的行业

好结局：
- AI协作专家 → 成为会用AI的高手
- 自由职业者 → 找到稳定的自由工作
- 小老板 → 带着AI工具接项目

完美结局：
- AI公司创始人 → 做出被别人恐惧的AI
- 不可替代的人 → 找到AI做不到的事
- 时代赢家 → 在AI时代活得很好
```

## 六、事件池（肉鸽要素）

### AI相关事件

```
随机事件：
- OpenAI发布新模型 → AI能力+1
- 某公司大规模裁员 → 工作机会-1
- AI出现Bug → 临时机会
- AI监管政策出台 → 某领域暂时安全
- 新AI工具发布 → 某技能贬值
- AI行业泡沫 → 临时机会
- AI替代速度加快 → 压力+1
- AI突破瓶颈 → 全面威胁
```

### 市场事件

```
- 某行业需求暴涨 → 收入机会
- 经济下行 → 收入下降
- 新兴行业出现 → 转型机会
- 行业洗牌 → 危机+机会
```

### 个人事件

```
- 身体报警 → 需要休息
- 家人需要钱 → 压力+1
- 朋友被裁 → 焦虑+1
- 找到导师 → 学习加速
- 遇到机会 → 转型可能
```

## 七、数值设计

### 核心数值

```
玩家属性：
- 技能等级（每个技能0-10）
- 收入（每月）
- 压力值（0-100）
- AI威胁度（0-100）
- 自由时间（每周几小时）

AI属性：
- 能力值（按领域）
- 成本（比人类便宜多少）
- 速度（比人类快多少）
- 质量（比人类好多少）

市场属性：
- 各技能的需求度
- 各技能的报酬
- 竞争程度
```

### 平衡设计

```
难度曲线：
- 前期：AI还不太强，有时间学习
- 中期：AI快速进化，压力增大
- 后期：必须找到出路，否则失败

成长曲线：
- 技能学习需要时间
- 但AI进化更快
- 必须做出选择

策略深度：
- 学习AI不擅长的？→ 但AI也在进化
- 学会用AI？→ 但大家都在学
- 转型？→ 需要时间和成本
- 创业？→ 风险大
```

## 八、总结

### 玩家玩的是什么？

```
在AI时代找出生路的策略游戏

核心体验：
- 焦虑感（AI威胁）
- 选择压力（学什么、做什么）
- 策略深度（多种出路）
- 成就感（找到出路）
```

### 三大核心机制

```
1. 玩家成长
   - 技能学习
   - 职业转型
   - 跨局继承

2. AI进化
   - 随机进化路径
   - 阶段性威胁
   - 不可预测性

3. 危机压力
   - 时间压力
   - 经济压力
   - 心理压力
```
