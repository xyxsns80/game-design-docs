# 题材范围：大而全 vs 小而精

> 讨论：模拟人生这种大题材太难处理，聚焦小事是否更容易出彩？

---

## 一、问题分析

### 大题材的困境

**以"人生模拟"为例：**

| 问题 | 说明 |
|------|------|
| **内容量巨大** | 人生能做的事太多了，怎么写得完？ |
| **难以平衡** | 不同玩家关注点不同，很难都满足 |
| **容易空洞** | 面面俱到=面面不到 |
| **迭代困难** | 改一处可能影响全局 |

```
人生模拟要包含：
- 出生设定（家庭、天赋、外貌...）
- 学生时代（学习、交友、恋爱...）
- 工作（职业、升职、跳槽...）
- 婚姻（相亲、结婚、生子...）
- 晚年（退休、养老...）
- 各种随机事件...

→ 太大了，很难做深
```

### 小题材的优势

**以"相亲"为例（《纸上谈亲》）：**

| 优势 | 说明 |
|------|------|
| **范围明确** | 就讲相亲这一件事 |
| **容易做深** | 把相亲的细节做透 |
| **玩家预期清晰** | 来就是相亲的 |
| **迭代简单** | 加新相亲对象/事件就行 |

```
相亲模拟只需要：
- 人物设定（标签系统）
- 相亲过程（选择、隐藏、揭示）
- 结局（结婚/失败）

→ 范围小，可以做到极致
```

---

## 二、大题材 vs 小题材对比

| 维度 | 大题材（人生） | 小题材（相亲） |
|------|----------------|----------------|
| **内容量要求** | 极大 | 小 |
| **开发周期** | 长 | 短 |
| **打磨深度** | 难 | 易 |
| **玩家预期** | 高（什么都想体验） | 低（就体验这一件事） |
| **满足感** | 难（总感觉缺什么） | 易（把一件事做好就行） |
| **迭代方向** | 不清晰 | 清晰 |
| **MVP验证** | 难 | 易 |

## 三、"小切口"策略

### 核心思路

```
从一个大话题中，切出一个具体的场景/阶段
```

### 切分示例

| 大题材 | 可切分的小题材 |
|--------|----------------|
| **人生** | 相亲、面试、高考、买房、养娃 |
| **职场** | 第一天入职、面试、年会、加班 |
| **恋爱** | 第一次约会、见家长、求婚 |
| **校园** | 高考、宿舍生活、社团 |
| **修仙** | 炼丹、渡劫、收徒 |

### 《纸上谈亲》的成功

```
大题材：人生/婚恋
小切口：相亲（人生的一个具体阶段）

为什么成功：
1. 范围明确 - 就讲相亲
2. 共鸣强烈 - 相亲是普遍痛点
3. 细节丰富 - 可以把相亲的各种场景做透
4. 扩展方便 - 加新相亲对象/事件
```

### 《人生重开》的策略

```
大题材：人生
策略：用"随机事件"代替"完整模拟"

不是模拟人生的每个细节，
而是用随机事件串联，
让玩家脑补中间的过程。
```

## 四、如何选择小切口？

### 选择标准

| 标准 | 说明 |
|------|------|
| **共鸣强** | 这个小场景是不是很多人经历过？ |
| **冲突多** | 这个场景有没有足够的选择和冲突？ |
| **话题性** | 这个场景是不是容易引发讨论？ |
| **可扩展** | 能不能持续加新内容？ |

## 五、小切口的深度设计

### 把一件事做透

**以"相亲"为例，可以做深的维度：**

1. **人物维度**
   - 不同类型的相亲对象
   - 隐藏标签的揭示
   - 好感度系统

2. **场景维度**
   - 咖啡厅、餐厅、公园...
   - 不同场景有不同事件

3. **选择维度**
   - 说什么话
   - 穿什么衣服
   - AA还是请客

4. **结局维度**
   - 成功/失败
   - 不同类型的成功
   - 隐藏结局

### 公式

```
小切口 + 多维度深挖 = 小而精
```

---

## 六、结论

### 🏆 小题材 > 大题材（对文字游戏）

**原因：**
1. 开发成本低
2. 容易做深做透
3. 玩家预期清晰
4. 迭代方向明确
5. MVP验证快

### 最佳策略

```
从大话题中切出一个具体场景 + 把这个场景做深做透
```

### 示例

| 不要做 | 要做 |
|--------|------|
| 人生模拟器 | 相亲模拟器 |
| 职场模拟器 | 面试模拟器 |
| 恋爱模拟器 | 第一次约会模拟器 |
| 校园模拟器 | 高考模拟器 |
