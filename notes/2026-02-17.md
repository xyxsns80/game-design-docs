### 主题1：借鉴《纸上谈亲》的设计理念

#### 讨论内容

1. **趣味文本的重要性**
   - 暴击/击杀/胜利文本应该随机化
   - 幽默感能让重复游玩不枯燥
   - 已实现于《肉鸽征途》

2. **信息揭示机制**
   - 敌人信息不应该一开始全部显示
   - 分阶段揭示：数量 → 属性 → 详情
   - 已实现于《肉鸽征途》

3. **继承系统（待讨论）**
   - 章节间保留部分进度
   - 让玩家感觉到成长积累

### 主题2：可持续盈利游戏的核心特征

#### 问题
什么样的游戏是一款可以持续盈利的好游戏？他们应该具有什么特点？

#### 结论

**核心公式：**
```
可持续盈利 = 高复玩性 × 多层变现 × 碎片化友好 × 自传播能力
```

**关键特征：**
1. **高复玩性** - 肉鸽、随机、收集、排行榜
2. **多层变现** - 广告+小额+中额+大额付费
3. **碎片化友好** - 单局3-5分钟，随时暂停
4. **自传播能力** - 截图有趣、话题性、挑战性

**适合小团队的方向：**
- 文字模拟（成本低）
- 肉鸽卡牌（复玩性高）
- 放置挂机（变现能力强）
- 超休闲游戏（传播快）

**最佳公式：**
```
低成本高收益 = 文字/简单美术 + 肉鸽机制 + 广告+内购 + 社交传播
```

详细分析见：[docs/mechanics/可持续盈利游戏特征.md](../docs/mechanics/可持续盈利游戏特征.md)

### 主题3：文字游戏与其他游戏的设计差异

#### 问题
文字游戏和其他游戏的从设计而非表现层面，主要区别是什么？

#### 结论

**五大核心差异：**

| 维度 | 文字游戏 | 其他游戏 |
|------|----------|----------|
| 信息载体 | 文字（抽象符号） | 图像/声音（具象感知） |
| 想象空间 | 玩家自行脑补 | 开发者定义 |
| 核心交互 | 选择驱动 | 操作驱动 |
| 内容核心 | 叙事+数值 | 沉浸+体验 |
| 迭代速度 | 极快（改文字） | 慢（改资产） |

**文字游戏的设计要点：**
1. 文案即核心（每句话都要有趣）
2. 选择即玩法（每个选择要有意义）
3. 数值即深度（属性、装备、概率）
4. 分支即内容（多结局、隐藏内容）
5. 留白即想象（不要写太满）

**文字游戏公式：**
```
好玩的文字游戏 = 有趣文案 + 有意义选择 + 深度数值 + 想象空间
```

详细分析见：[docs/mechanics/文字游戏设计差异.md](../docs/mechanics/文字游戏设计差异.md)

### 主题4：文字游戏题材和玩法的选择

#### 问题
文字游戏题材和玩法的选择主要应考虑什么？以及备选方案？

#### 结论

**题材选择5要素：**
1. 目标用户匹配 - 谁玩？什么时候玩？
2. 话题性和传播力 - 有共鸣、有话题、有梗
3. 内容扩展性 - 可以持续添加新内容
4. 玩法适配性 - 数值系统是否自然
5. 风险和合规 - 是否敏感

**玩法选择5要素：**
1. 核心循环设计 - 选择→结果→反馈→新选择
2. 选择设计 - 无正确答案、有惊喜
3. 数值系统深度 - 属性制约、随机性
4. 目标和反馈 - 短中长期目标明确
5. 复玩性设计 - 多结局、随机、收集

**题材备选TOP5：**
| 题材 | 传播力 | 扩展性 | 案例 |
|------|--------|--------|------|
| 相亲/婚恋 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 《纸上谈亲》 |
| 职场/打工 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 《打工模拟器》 |
| 人生/命运 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 《人生重开》 |
| 修仙/玄幻 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 《修仙模拟器》 |
| 商业/经营 | ⭐⭐⭐ | ⭐⭐⭐⭐ | 《商业大亨》 |

**小团队首选组合：**
```
职场/人生题材 + 肉鸽骰子玩法 + 微信传播 + 广告变现
```

详细分析见：[docs/mechanics/题材和玩法选择.md](../docs/mechanics/题材和玩法选择.md)

### 主题5：亲身体验 vs 幻想实现

#### 问题
玩家有亲身体会的和玩家无法在现实中达到的这两种方向，哪种方向更好一点？

#### 结论

**两种方向对比：**

| 维度 | 亲身体验 | 幻想实现 |
|------|----------|----------|
| 代入感 | ⭐⭐⭐⭐⭐ | ⭐⭐ |
| 话题性 | ⭐⭐⭐⭐⭐ | ⭐⭐ |
| 新鲜感 | ⭐⭐ | ⭐⭐⭐⭐⭐ |
| 付费意愿 | ⭐⭐ | ⭐⭐⭐⭐⭐ |
| 传播成本 | 低 | 高 |
| 开发难度 | 低 | 高 |

**🏆 推荐：亲身体验类**

理由：
1. 小团队优势更大（文案取材生活）
2. 更适合微信生态（话题性传播）
3. 开发风险更低（快速验证迭代）

**最优公式：**
```
亲身体验的题材 + 荒诞有趣的反转 = 低成本高传播
```

示例：
- 《纸上谈亲》= 真实相亲 + 离谱对象 + 荒诞结局
- 《人生重开》= 真实开局 + 荒诞事件 + 意外结局

**选择指南：**
- 小团队/微信小游戏 → 亲身体验类
- 大团队/长线运营 → 幻想实现类

详细分析见：[docs/mechanics/题材方向对比.md](../docs/mechanics/题材方向对比.md)
